# Demo Operator Walkthrough Checklist

Use this sequence for a consistent, low-friction manual demo of the current MVP.

## 1. Start State

- Confirm the header shows Tampa Bay scope and freshness.
- Confirm Data Readiness is visible (live/degraded sources and planning baseline separation).
- Confirm no active selection state confusion (no stale selected case when dashboard is empty).

## 2. Submit Incident

- Open **Submit Incident Report** utility.
- Enter incident description and county scope.
- Optionally generate decision guide for preparatory framing.
- Click **Analyze Incident**.

## 3. Explain Global Picture First

- Read KPI cards as global metrics (not selected-case filtered).
- Call out that planning metrics are non-live and separated from live evidence.
- Use Operational Snapshot for concise global severity/region recap.

## 4. Move to Selected-Incident Operations

- Select a fused case card to focus all operational panels.
- Verify Current Operational Focus updates (what changed, risk, next actions, supporting evidence).
- Click an alert to demonstrate bidirectional selection sync.

## 5. Evidence and Planning Separation

- In Live Evidence panel, show focused incident observations only as live evidence.
- In Planning Baseline panel, show preparation actions and non-live labels.
- Use reason tags to explain why each planning action exists.

## 6. Map Interpretation

- Explain legend categories (live evidence vs planning baseline non-live).
- Show selected-incident prominence over contextual overlays.
- Point out map freshness and degraded-state messages when present.

## 7. Close with Actionability

- Summarize: what is disrupted, what resources are affected, what evidence supports it, and what to do next.
- If partial/degraded banner appears, state that guidance is still usable but requires operator verification.
